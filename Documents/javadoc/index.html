Dolby Audio Plug-in for Cordova

This Cordova 3 plug-in provides access to Dolby® Audio Processing.

Applications can use the APIs to control Dolby Audio Processing background service integrated into an Android™ device. The plug-in uses less than 1k memory, and the audio processing adds only 21 ms of delay at 48 kHz, 23ms at 44.1 kHz for audio output.

Version 1.0.0.0

Installation

To install the plug-in, use the Cordova 3 Command-line Interface:

Cordova plugin add <path to Cordova plugin>

Supported Platforms

  • Amazon Fire OS 3.0 and later versions
  • Android 4.0 and later versions

Example

  1. Initialize Dolby Audio Processing on Cordova's deviceready event and specify the default profile.

    function bindEvents() {
    document.addEventListener('deviceready', onDeviceReady, false);
};
function onDeviceReady() {
    document.addEventListener("pause", onPause, false);
    document.addEventListener("resume", onResume, false);

    dolby.dap.initialize(dolby.DapProfile.GAME, onDsConnected, onFail);
};
function onDsConnected() {
    alert("Dolby Audio Processing is initialized.");
};
function onFail(err) {
    alert(err.message);
};

    NOTE: Dolby Audio Processing is switched on automatically after it is initialized successfully, even if it was in off status before being loaded. Call the initialize() API to initialize Dolby audio processing before you play any audio stream.

  2. Suspend the application Dolby Audio Processing session on Cordova's pause event when it goes to the background or minimized state.

    function onPause() {
    dolby.dap.suspendSession();
};

  3. Resume the application Dolby Audio Processing session on Cordova's resume event when it returns to the foreground.

    function onResume() {
    dolby.dap.restartSession(function () {
        dolby.dap.getSelectedProfile(function(value) {
            alert("proflie: " + value);
        }, null);
    }, null);
};

    NOTE:Correctly using suspendSession() API and restartSession() API prevents an application from changing the system-wide Dolby Audio Processing configuration.

  4. Release Dolby Audio Processing and restore system configuration to the state prior to application control on HTML body's onunload event. Actually the release() API is optional, the plug-in releases automatically when the Cordova application is shutting down..

    function release() {
    dolby.dap.release();
};

  5. Select a different Dolby Audio Processing profile when an application is in the foreground.

    dolby.dap.setProfile(dolby.DapProfile.MUSIC, null, null);

  6. Disable Dolby Audio Processing when an application is in the foreground. Best Practice: Do not turn off Dolby Audio Processing in an application lifecycle.

    dolby.dap.setEnabled(true, null, null);

    NOTE:

    • To make sure control Dolby Audio Processing successfully, the APIs (setProfile(), setEnabled(), getSelectedProfile(), etc) should be called only after a successful callback of initialize() API is invoked.
    • To implement the basic functionality, following steps 1-3 above makes your Cordova application leverage Dolby Audio Processing technology safely and correctly.

Known Issues

  • Dolby Audio Processing does not support Samsung Galaxy Tab3 8”(SM-T310) and 10.1”(GT-P5210) devices now.
