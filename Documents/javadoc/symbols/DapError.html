Field Detail
<static> DapError.ACTION_NOT_ALLOWED
This error returns when the action is not allowed temporarily, e.g. if the user tries to switch the profile when Dolby Audio Processing is turned off. We recommend first turning on Dolby Audio Processing, then switching the active profile. The dolby.dap.setEnable()/dolby.dap.setProfile() API returns this error code if dolby.dap.suspendSession() API has been called. --------------------------------------------------------------------------------
<static> DapError.DAP_UNAVAILABLE
This error returns when Dolby Audio Processing is not available on a device. The error indicates the onFail callback of initialize() API when Dolby Audio Processing is not available on a device.
<static> DapError.INVALID_PARAM
This error returns when when the input parameter is invalid.
<static> DapError.NOT_CONNECTED
This error returns when the background Dolby service is not connected. The initialize() API should be called if it occurs. If initialize() API is not invoked, the other APIs return this error . In an unusual scenario (e.g. Android OS memory is used up), if the background Dolby service is restarted, the error can occur before the initialize() API is called to reinitialize Dolby Audio Processing.
<static> DapError.RUNTIME_ERR
This error returns when a run-time error occurs, e.g. an Android OS or Dolby service internal error. The error indicates that Dolby Audio Processing is not working correctly.
